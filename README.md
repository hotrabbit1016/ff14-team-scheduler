# FF14 固定團排班

免登入的 FF14 零式 / 絕本固定團每週排班工具。團長建立隊伍後只要設定攻略內容；成員從自己的固定職位卡進入並填可參與時段，隊伍頁會在同一頁整理本週可練幾場、職位回覆狀態與 Discord 公告。

全站以台灣時間顯示。時間格固定 30 分鐘，推薦時段會用飯數顯示長度，例如 `4飯 = 2 小時`；排班流程本身仍使用「出團、場次、時段、填表、DC公告」等常用語境。

## 主要流程

- `本週可練場次`：依團員填的可參與時段計算本週能練幾場，每天最多取一場最佳交集。
- `回覆進度`：用 `MT / ST / H1 / H2 / D1 / D2 / D3 / D4` 顯示已填、未填與本週無法。
- `團員名單`：八格位置看板，空位置可直接點 `填 MT / 填 ST...` 進入填表。
- `回覆進度`：可直接複製隊伍頁連結，團員從自己的位置進去填。

## v2 重點

- 定位固定為零式 / 絕本 8 人固定團，不處理 4 人本或 24 人本。
- 團員職能使用固定團常見位置：`MT / ST / H1 / H2 / D1 / D2 / D3 / D4`。
- 舊資料的 `TANK/HEALER/DPS/FLEX` 會自動轉成 `MT/H1/D1/D4`，避免讀取失敗。
- 團員填表只保留 `本週無法出團` 與可出時段；晚到或只能打部分時間就直接填可用區間。
- 建立頁不要求本週目標場次，避免團長建立隊伍頁前就要先決定排幾場。
- 回覆進度以固定團 8 個位置為核心，已填時段或標記本週無法出團都算已回覆。
- 排班卡會以 `第 1 場 / 第 2 場` 顯示可出人數、缺席名單、職能缺口與時段飯數。
- `raid_plans` 保留在資料層以相容舊資料，但目前不出現在主流程。
- 介面使用 XIVAPI 圖示來源搭配自製 FF14 inspired 水晶終端風格，不使用官方 Logo。

## 開發

```bash
npm install --cache ./.npm-cache
npm run dev -- --port 5173
```

開啟 http://127.0.0.1:5173/

## 資料模式

若未設定 Supabase 環境變數，App 會使用瀏覽器 `localStorage`，方便本機驗證流程。這種模式無法跨瀏覽器或跨裝置同步。

要使用 Supabase：

1. 在 Supabase SQL editor 執行 `supabase/schema.sql`。
2. 建立 `.env.local`：

```bash
VITE_SUPABASE_URL=你的 Supabase Project URL
VITE_SUPABASE_ANON_KEY=你的 Supabase anon key
```

3. 重新啟動 dev server。

v2 依規劃使用公開讀寫 policy，任何拿到連結的人都可以加入、修改或刪除成員資料。
